学术论文 PDF 仿生阅读工具
最终需求文档（V1）
0. 项目定位（一句话）

这是一个面向学术论文 PDF 的在线仿生阅读工具，
将论文内容进行结构化重排为单栏文本，并对中文与英文分别应用合适的仿生阅读规则，以提升长文阅读的效率与专注度。

1. 支持对象与边界（必须写死）
1.1 支持的 PDF 类型

文本型学术论文 PDF

常见结构包括：

标题

作者

摘要

正文（单栏或双栏）

注释 / 基金说明

参考文献

1.2 明确不支持（V1 不做）

扫描件 PDF（不做 OCR）

杂志、宣传册、复杂图文混排文档

表格为主的技术报告

高度非标准版式（艺术排版）

说明：V1 的目标是稳定处理 80–90% 的标准学术论文，而不是覆盖所有 PDF。

2. 总体处理流程（不可更改）
PDF 上传
→ 文本与坐标抽取
→ 页面结构识别
→ 正文阅读顺序恢复
→ 单栏结构化重排
→ 中文仿生阅读处理
→ 英文仿生阅读处理
→ HTML 阅读输出

3. PDF 页面结构处理规则（学术论文专用）
3.1 文本抽取

使用 PDF 工具（如 pdf.js）抽取文本块

每个文本块至少包含：

文字内容

页面编号

坐标信息（x / y）

大致字号或高度信息（如可获取）

3.2 页面功能区识别（按语义，不按视觉）

每一页 PDF，需识别并区分以下功能区：

标题区

位于页面顶部

字号明显大于正文

可包含主标题 / 副标题

作者信息区

紧随标题

字号小于标题，大于正文或接近正文

摘要区

包含“摘要 / Abstract”关键词

通常为单栏文本

位于正文之前

正文区

主要阅读内容

可能为单栏或双栏

注释 / 基金说明 / 页脚

位于页面底部

字号较小

包含基金编号、页码等信息

3.3 正文阅读顺序恢复（核心规则）
以“每页”为独立单位处理正文区：

判断是否为双栏：

基于文本块 x 坐标分布进行聚类

若为双栏：

阅读顺序为：左栏自上而下 → 右栏自上而下

若为单栏：

阅读顺序为：自上而下

不要求还原原始排版，仅保证阅读顺序正确、连续。

3.4 注释与页脚处理

注释、基金说明、页码：

不得插入正文中

应收纳至对应页面正文之后

标记为「附注」

4. 输出结构（统一单栏）

所有内容最终输出为单栏滚动阅读结构：

标题
作者

摘要

正文（单栏，连续阅读）

【附注】
……

5. 中文仿生阅读规则（V1 · 定义）
5.1 核心原则

不做分词

不按“词”处理

仅依据句子边界进行加粗

目标是：给每个句子的阅读入口提供视觉锚点

5.2 句子边界定义（触发新句）

以下任意字符或结构，视为一个新句子的开始：

中文标点：
。 ！ ？ ； ： ， 、 

英文标点：
. ! ? ; : ,

结构边界：

换行符 \n

段落开头

注：是否将「逗号 / 顿号」作为句子边界，需作为可配置开关。

5.3 加粗规则

每个新句子开始时：

从句子起点向右扫描

跳过非正文字符（见 5.4）

对后续 前 N 个中文字符进行加粗

默认参数：

N = 4

可调范围：2–6

5.4 必须跳过的字符（不计入加粗）

以下字符在句子起点后需跳过，不参与加粗与计数：

空白字符（空格、制表符）

引号类：

“ ” ‘ ’ 「 」 『 』

括号类：

（ ） ( ) [ ] 【 】

书名号：

《 》

破折号：

— – -

其他装饰符号（如 · …）

5.5 中文字符定义

每一个 Unicode 汉字视为 1 个字符

不跨句、不跨段加粗

6. 英文仿生阅读规则（V1）

采用成熟的开源英文 Bionic Reading 实现

按英文单词处理

加粗单词前 40–50%

不自行发明英文规则

7. 功能范围（V1）
7.1 必须实现

PDF 上传

学术论文文本抽取

双栏 / 单栏正文顺序恢复

中文句首仿生阅读加粗

英文仿生阅读加粗

HTML 在线阅读

7.2 明确不做

PDF 导出

OCR光学字符识别

用户账号系统

支付系统

原版式还原

8. 成功验收标准（不可模糊）

V1 版本完成的标准是：

能成功上传学术论文 PDF

正文阅读顺序正确（双栏不乱）

中文每句前 N 字加粗生效

英文仿生阅读生效

页面可连续阅读，不影响段落结构

链接分享给他人可正常使用

9. 产品态度声明（非常重要）

本工具的目标是：

提升学术论文的“可读性”与“持续阅读能力”，
而不是复刻纸面排版或实现出版级 PDF 编辑。
